# Narrative plane

Hand-authored, audience-shaped docs live here. This plane is never auto-generated, just as the synthesis plane (`SYNTHESIS.md`) is never hand-edited; the two must not clobber each other.

What belongs here:

- **Implications and recommendations.** The synthesis states what the evidence shows; this plane says what it might mean and what could be done about it, framed as possible approaches for the reader to weigh rather than directives.
- **Claims that aren't yet traceable to `data/`.** Hunches, presumptions, and open questions are welcome here, labeled as such.
- **Audience-shaped tellings.** One evidence base can feed several docs cut for different readers (an executive brief, a practitioner deep-dive, a buyer-facing view). Name the docs at seeding and keep each in its audience's register.

Ground rules:

- Cite the synthesis or the evidence store when a narrative claim rests on it, so a reader can trace the chain.
- When narrative work surfaces a factual gap, send it back through the research loop (a HANDOFF priority, a new pass) rather than patching the synthesis by hand.
- Keep the voice conventions from the topic CLAUDE.md.

This directory starts empty apart from this README. That is deliberate: a starter doc here would invite auto-generating the one plane that must stay hand-authored.
